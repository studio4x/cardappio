# PROMPTS.md — Cardappio

## Purpose

This document provides ready-to-use prompts for generating screens and visual directions for **Cardappio** inside Stitch.

It should be used together with:
- `DESIGN.md`
- `SCREENS.md`

The prompts are written in English to work better with design-generation tools.

---

# 1. Master Prompt

Use this prompt first to establish the general product direction.

## Master Product Prompt

> Create a mobile-first interface for an app called Cardappio, focused on weekly meal planning. The product should feel clean, modern, light, welcoming, practical, and highly usable. It should not feel like a generic recipe catalog, but like a home meal-planning and shopping-list assistant. Prioritize excellent mobile usability, card-based layouts, clear hierarchy, generous spacing, intuitive navigation, and a friendly but functional visual identity. Use a fresh green as the main action color, light neutrals as the foundation, and subtle warm accents such as soft terracotta or muted orange for food-related highlights. The experience should communicate routine simplification, time saving, domestic organization, and less stress around deciding what to cook during the week.

# 2. Visual Identity Prompts

## Prompt — Brand Direction

> Create a visual identity direction for Cardappio, a weekly meal planning app for home use. The brand should feel practical, organized, friendly, modern, and trustworthy. It should communicate routine simplification, food planning, shopping organization, and ease of use. Avoid an old-fashioned recipe-blog style, avoid dense SaaS dashboard styling, and avoid overly playful food branding.

## Prompt — Color Direction

> Create a color palette for a weekly meal planning app called Cardappio. Use a fresh green as the main brand and action color, supported by soft warm accents like muted orange or terracotta for food-related highlights. Use very light neutrals as the interface base, with soft borders and highly readable text contrast. The palette should feel fresh, calm, domestic, and organized.

## Prompt — Typography Direction

> Create a typography direction for Cardappio, a mobile-first meal planning app. Use clean sans-serif typography with strong readability on mobile, simple and modern headings, soft neutral body text, and a practical, organized feel. Prioritize usability and clarity over decorative personality.

## Prompt — Component Style Direction

> Create a component style direction for Cardappio. The app should rely on cards, meal slots, recipe previews, shopping checklist rows, chips, badges, and simple action buttons. The UI should feel modular, airy, and easy to scan, with strong hierarchy and clear actions.

# 3. Public Website Prompts

## Prompt — Public Landing Page

> Create a landing page for Cardappio, a weekly meal planning app. The page should explain the product clearly and convert visitors into users. Include a clean header with logo and CTAs, a hero section with a strong value proposition, an app preview, a short “how it works” section, key benefits, a section about automatic shopping list generation, a pricing teaser, FAQ preview, final CTA, and footer. The style should be clean, modern, welcoming, mobile-first, and visually organized.

## Prompt — How It Works Page

> Create a “How It Works” page for Cardappio. The page should explain the weekly user flow in a simple, step-by-step format: choose the days, select lunch and dinner recipes, build the week, generate the shopping list, and use the recipes during the week. Use clean visuals, step cards, soft illustrations or icons, and a strong CTA to sign up.

## Prompt — Pricing Page

> Create a pricing page for Cardappio that compares free and premium plans clearly. The page should feel friendly, clean, and conversion-oriented. Highlight the value of premium features without looking aggressive. Include plan cards, benefit comparison, a recommended option, and simple CTAs.

## Prompt — FAQ / Support Page

> Create an FAQ and support page for Cardappio. Use a clean, calm layout with expandable FAQ items, common questions about weekly planning, shopping lists, recipes, subscriptions, and account usage. Include a support/contact CTA.

# 4. Authentication & Onboarding Prompts

## Prompt — Login / Signup Screen

> Create a mobile-first login and signup screen for Cardappio. The screen should feel simple, calm, welcoming, and trustworthy. Include email, password or simplified access method, primary CTA, secondary links for sign up and forgot access, and optional soft visual support. Prioritize clarity and low friction.

## Prompt — Onboarding Flow

> Create a short onboarding flow for Cardappio, a weekly meal planning app. The purpose is to collect just enough information to personalize the user experience without feeling heavy. Include steps for household size, lunch/dinner preference, number of planning days, food restrictions or preferences, and main goal such as saving time, saving money, eating better, or organizing family meals. Use a mobile-first, calm, guided, multi-step layout with a subtle progress indicator.

## Prompt — Onboarding Step Screen

> Create one step of a mobile-first onboarding flow for a meal planning app. Show a short question, selectable options using chips or cards, a clear next button, and a subtle progress bar. The interface should feel simple, light, and encouraging.

# 5. Core App Experience Prompts

## Prompt — Dashboard / App Home

> Create a dashboard for Cardappio, a weekly meal planning app. Show the current week summary, a primary CTA to create or edit the week, quick access to the shopping list, favorites, and lightweight recipe suggestions or featured collections. ## Prompt — Weekly Planner Empty State

> Create the empty state of a weekly meal planner for a meal planning app. Show a clear message that the user has not built the week yet, with simple visual guidance and a strong CTA to start planning. The interface should feel calm, practical, and encouraging.

## Prompt — Weekly Planner Filled State

> Create the filled state of a weekly meal planner for Cardappio. Show several days with lunch and dinner meal slots already populated by selected recipes. Include replace and remove actions, visual consistency across days, and CTAs to save or review the shopping list.

## Prompt — Weekly Meal Plan Summary Screen

> Create a weekly meal plan summary screen for Cardappio. Show the completed week clearly with each day, lunch and dinner entries, quick actions to open recipes, edit the week, print/share, and go to the shopping list. The interface should be easy to scan and very usable on mobile.

> Create a reusable recipe card component for Cardappio. The card should show a food image, recipe title, prep time, difficulty, cost, and a couple of useful tags. It should feel modern, clean, appetizing but not overly editorial, and optimized for mobile list or grid use.

## Prompt — Recipe Detail Screen

> Create a recipe detail screen for Cardappio. Include recipe image, title, subtitle, badges for prep time, difficulty, cost, and servings, a clear ingredient list, step-by-step instructions, notes or tips, a favorite button, and an add-to-plan CTA. Prioritize readability and cooking usability over decorative content.

## Prompt — Recipe Detail Cooking Mode

> Create a cooking-friendly recipe detail layout for a mobile meal planning app. Prioritize large readable text, strong hierarchy for ingredients and steps, easy scrolling, and a practical interface for real kitchen use.
